üìä QR Code Structure & Validation Logic

QR Code Format (14 Characters)

Position 1: Site Identifier
Position 2: Line ID (A-Z) - Must match Batch Line
Position 3-12: Batch Number (10 alphanumeric)
Position 13: Mould Alphabet (A-Z)
Position 14: Serial Number (0001-9999)

‚úÖ PASS (GREEN)

Format valid; Line matches; In-range; Not duplicate ‚Üí Routed to PASS bin; Green LED ON

‚ö†Ô∏è DUPLICATE (YELLOW)

QR already scanned in current session ‚Üí Counted as duplicate; Yellow status; stays out of bin flow

‚ùå REJECT (RED)

Invalid format; Line mismatch; Out of batch range ‚Üí Routed to REJECT bin; Red LED ON

üîé Scan

üîê 4-Level Validation Process

1

Format Validation

Check: QR must be exactly 14 alphanumeric characters
Rejects: Wrong length, special characters, spaces
Result: RED light if failed

2

Line ID Validation

Check: Position 2 of QR must match configured Batch Line (A-Z)
Rejects: Cartridges from different production lines
Result: RED light if Line mismatch

3

Batch Range Validation

Check: QR must fall within configured mould ranges (Start to End QR)
Rejects: Cartridges from different batches
Result: RED light if out of range

4

Duplicate Detection

Check: QR not previously scanned in current session
Prevents: Re-scanning same cartridge multiple times
Result: YELLOW light if duplicate, GREEN if all checks pass
